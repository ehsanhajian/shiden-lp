Building the premier multi-chain smart contract platform on Kusama Network

Shiden Network is a multi-chain decentralized application layer on Kusama Network. Kusama Relaychain does not support smart contract functionality by design - Kusama Network needs a smart contract layer. This is where Shiden Network comes in. Shiden supports Ethereum Virtual Machine, WebAssembly, and Layer2 solutions from day one. The platform supports various applications like DeFi, NFTs and more.

What Makes Shiden Unique?

Multi Chain

Building bridges to various blockchains. (Ethereum, Secret Network, Cosmos and so on)

Basic Income for Developers

SDN token holders can stake their tokens on favorite dApps so that both nominators and the dApps developer can earn SDN tokens.

Multi Virtual Machine

Shiden natively supports both EVM and WASM. Ethereum compatibility is just a day1 for Kusama Parachains.

How to Deploy dApps?

  1. Visit our doc

    Learn how Shiden Network works and how to deploy smart contracts.

  2. Use EVM or WASM

    Shiden Network supports both EVM and WASM. Choose a virtual machine that is suitable for your team and your use case.

  3. Compile the contract and deploy on Dusty

    If you are trying to deploy Solidity smart contracts, you have 2 ways to compile your smart contract: using Ethereum tools, or using Solang (a Solidity to WASM compiler). After compiling your contract, you can deploy it on our testnet, Dusty Network.

  4. Deploy on Shiden

    After successful development on Dusty, you can now deploy your contract on Shiden.
